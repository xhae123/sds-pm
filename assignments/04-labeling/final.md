## 1. 프로젝트 주제

Zan Sol — 집중 세션 중 딴짓을 자동 감지하고 유저를 긁는 잔소리로 즉시 개입하는 개인용 자기조절 앱.

## 2. 과제 목적 (우리 맥락)

V1은 규칙 기반이므로 별도 ML 모델 없음. 잔소리 선택은 4축(잔소리 무시 후 딴짓 지속 단계·앱 카테고리·현재 모드·디데이) 공간에서 최적점 탐색 문제이며, 각 축 정의역이 4개만 되어도 256(4^4) 조합. 전부 사람이 설계 불가 → **대표 조합 10개 직접 라벨링하여 판단 기준 명시화**.

## 3. Labeling Examples

- **구글시트 링크:** https://docs.google.com/spreadsheets/d/14tub6E1RnEArBwZdwPCpfbtbE7ij_wYGfAWQPP8_Vs/edit?usp=sharing
- 10개 조합을 (단계·카테고리·모드·디데이·부여 라벨·이유·판단 기준) 7개 열로 라벨링

### 라벨링 후 발견한 4가지 핵심 패턴

1. **디데이가 가장 강력한 조절 변수.** D-3이면 거의 모든 카테고리에서 강력 또는 공감으로, D-30+이면 대부분 제안 또는 개입 보류로 내려감.
2. **앱 카테고리별 "몰입 깊이" 차이.** 영상/게임은 이탈 어려워 빠르게 강도 올려야, SNS/메신저는 단위 소비 짧아 공감형이 더 효과적.
3. **휴식 모드에서는 개입 정당성이 약해진다.** 같은 유튜브라도 공부 모드 vs 휴식 모드에서 잔소리 강도 크게 달라야. 휴식 모드의 강한 개입은 앱 삭제 리스크.
4. **메신저는 특수 카테고리.** 상대방 있는 소통이라 일방적 차단 시 사회적 비용. 다른 카테고리와 다른 접근 필요.

### 애매하거나 어려웠던 점

1. **카카오톡은 딴짓인가?** 친구와 잡담은 딴짓, 과제 관련 대화도 카카오톡. "사용자 지정 리스트"로 해결하지만 같은 앱 내 용도 혼재는 V1에서 해결 불가.
2. **휴식 모드에서 개입을 해야 하는가?** 사용자가 스스로 휴식 선언한 상태. 시스템 잔소리 시 "왜 쉬지도 못하게 해?" 반발. 휴식 모드는 빈도·강도 크게 낮추거나 시간 제한만 알려주는 게 적절할 수 있음.
3. **"제안"과 "공감"의 경계 모호.** "막막하지? 자료 하나만 찾아봐"는 공감인가 제안인가. 문구 수준에서 두 스타일이 섞이는 경우 많음.

## 4. Similar Public Implementations

### 사례 1: NudgeRank (ACM KDD 2024, 헬스케어 맥락 기반 nudge)

- 링크: https://dl.acm.org/doi/10.1145/3637528.3671562
- 유사점: 도메인 다르지만 I/O 구조 거의 동일. "사용자 현재 context" → "가장 효과적인 nudge 선택". 우리 4축이 이 논문의 context에 해당. 110만 명 운영 + 걸음 수 6.17% 증가 실증.
- 참고점: context → feature 인코딩 방식, nudge 효과 측정(=우리 복귀율)을 V2 설계 레퍼런스로.

### 사례 2: Duolingo 잔소리 푸시 (에스컬레이션 톤 설계)

- 링크: https://blog.duolingo.com
- 유사점: "가볍게 → 감정적으로 → 죄책감 유발" 에스컬레이션 구조. 방향 반대(안 올 때 잔소리 vs 딴짓할 때 잔소리)지만 메커니즘 동일.
- 참고점: 리텐션 성과 있는 검증 사례. "어디서부터 앱 삭제 위험이 커지는가" 임계점 간접 추정.

## 6. 종합 정리

### 초기 휴리스틱
잔소리 선택은 4축 공간 최적점 탐색. V1은 라벨링 10개 대표 조합을 규칙으로 코딩하여 문구 풀 설계. 핵심: **"디데이가 가장 강력한 조절 변수"**, **"앱 카테고리별 몰입 깊이 다름"**.

### 예상 challenge
1. 제안/공감/강력 경계 모호 → 명확한 분류 기준 필요
2. 휴식 모드 개입 정당성 약함 → 빈도/강도 하한 기준 미확립
3. 메신저 특수성(소통) → 다른 로직 필요
4. 256 조합 중 10개만 라벨링 → 나머지 일반화 규칙 필요

### 접근 방법 유지 가능성
유지. V1 규칙 기반 + V2 Contextual Bandit으로 나머지 공간 자동 탐색 경로 유효. NudgeRank가 "맥락 기반 개인화 메시지 추천"의 실증 예시.

### 추가 조사 필요
1. 휴식 모드 개입 전략 학술 근거
2. 앱 카테고리 자동 분류 (Google Play Store API 등)
3. 전문가 인터뷰(학습상담사·고교 교사)로 라벨링 가설 검증

## 수업 원칙 매핑

- **Step 6 (알고리즘이 되어보기):** 전문가 개입 절차 + 우리 직관을 규칙으로 옮기는 작업 자체가 Step 6의 실습.
- **Step 8 (framing 변경):** "256 조합 전부 설계" 대신 "10개 대표 + V2 탐색" framing으로 전환. Step 8 직접 적용.
- **"주제-데이터-모델이 논리적으로 맞물림":** 라벨링이 곧 데이터, 규칙이 곧 V1 모델, V1 운영이 V2 데이터를 낳는 구조.
