# 앱·웹 개발 프로젝트를 위한 UX 설계

**프로젝트:** Zan Sol (잔솔) — 집중 세션 중 딴짓을 자동 감지하고, 잔소리로 즉시 개입하는 개인용 자기조절 앱 (Android, V1 = 규칙 기반)

**팀:** 김우진(PM) · 유준혁 · 이호준 · 김민성

**날짜:** 2026-04-28

---

## 1. Information Architecture

### 1-1. 주요 사용자 유형

V1은 **단일 유형: 개인 사용자(학생 본인)**. 팀/관찰자/관리자 역할 없음.

### 1-2. 메뉴 트리 구조

```
Zan Sol
│
├── 홈  [P0]
│   ├── 오늘 카드 (오늘 TODO · 어제 복귀율)
│   ├── 세션 시작 버튼  ──→ (세션 진행으로 진입)
│   └── 익명 카운터 "지금 N명 집중 중"  [P2]
│
├── 세션  [P0]   ※ 진입은 홈에서만
│   ├── 세션 진행 화면 (경과 시간 · 현재 TODO · 마이크로 태스크)
│   └── 잔소리 오버레이  ※ 시스템 권한으로 OS 레이어 위에 표시
│       ├── 1차 — 행동 제안 (즉시)
│       ├── 2차 — 공감 + 난이도 낮추기 (30초 무시 시)
│       └── 3차 — 압박 (2분 무시 시)
│
├── 할 일  [P0]
│   ├── TODO 목록 (D-N 정렬)
│   ├── TODO 추가 / 마감 등록
│   └── 마이크로 태스크 ("한 줄만 써봐" 류) 자동 생성
│
├── 리포트  [P1]
│   ├── 일별 리포트 (집중 시간 · 딴짓 시도 · 복귀율)
│   └── 주별 리포트 (단계별 복귀율 · 문구별 복귀율)
│
└── 설정  [P0]
    ├── 딴짓 앱 등록 / 자동 감지 후보 승인
    ├── 잔소리 강도 (문구 톤 선택)
    ├── 학기 일정 (사용자 직접 입력 — freshness 유지 주체)
    └── 권한 관리 (AccessibilityService · SYSTEM_ALERT_WINDOW)
```

**범위 표기:** P0 = V1 필수 / P1 = V1 확장 / P2 = 후순위 / V2 = 차기 버전 (현재 트리에 없음)

### 1-3. 설계 의도

- **홈 → 세션이 1탭 안에 닿음.** 핵심 루프가 가장 짧은 경로.
- 리포트·설정은 보조 메뉴로 분리 (사용 빈도 낮음).
- **잔소리 오버레이는 메뉴 구조 외부에서 트리거됨** — 시스템 권한(AccessibilityService + SYSTEM_ALERT_WINDOW)으로 다른 앱(인스타·유튜브) 위에 그려진다. IA 표상 "세션" 하위지만 실제로는 OS 레이어 위에 떠 있다. 이게 우리 IA의 가장 특이한 지점이다.
- 비슷한 기능 중복 없음: TODO 등록은 "할 일"에만, 권한 관리는 "설정"에만.

## 2. User Flow

### 2-1. 선택한 핵심 사용자 목표

**"집중 세션을 시작하고, 딴짓을 시도했을 때 잔소리 개입을 받아 원래 세션으로 복귀한다."**

이 목표를 고른 이유: 우리 서비스의 Product Goal("그 순간 즉시 개입하여 복귀") 자체가 이 흐름이다. 다른 후보(온보딩, TODO 등록, 리포트 보기)는 보조 흐름.

### 2-2. 흐름도

```
[홈 화면]
  ↓ [세션 시작] 버튼
[TODO 선택 (오늘 등록된 항목 중 1개)]
  ↓ [시작]
◇ AccessibilityService 권한 있는가?
  ├─ NO → [권한 안내 화면] → 설정 이동 → 다시 시도          (예외 1)
  └─ YES ↓
[세션 진행 화면 — 타이머 시작, Foreground Service]
  ↓
◇ 딴짓 앱 실행 감지? (Reactive <0.1초 + 30초 폴링)
  ├─ NO → 세션 계속 → 정상 종료 → [세션 완료 리포트]
  └─ YES ↓
[1차 잔소리 오버레이 — "자료 하나만 찾아봐" (행동 제안)]
  ↓
◇ 사용자 반응?
  ├─ 복귀 (집중 앱 전환) → [세션 진행 화면 복귀] → 복귀율 기록
  └─ 무시 30초 ↓
[2차 잔소리 오버레이 — "막막하지? 한 줄만 써봐" (공감 + 난이도 낮추기)]
  ↓
◇ 사용자 반응?
  ├─ 복귀 → [세션 진행 화면 복귀] → 단계별 복귀율 기록
  └─ 무시 2분 ↓
[3차 잔소리 오버레이 — "내일의 너한테 미안하지 않을까?" (압박)]
  ↓
◇ 사용자 반응?
  ├─ 복귀 → [세션 진행 화면 복귀]
  └─ 끝까지 무시 → 세션 실패 기록 → [세션 완료 리포트(실패 표시)]   (예외 2)

[세션 완료 리포트]
  ├─ 집중 시간·딴짓 시도 횟수·단계별 복귀율 표시
  └─ [다음 세션 시작] 또는 [홈으로]
```

### 2-3. 의사결정 지점

1. AccessibilityService 권한 부여 여부
2. 실행 앱이 사용자가 등록한 딴짓 앱 리스트에 있는가
3. 1차 잔소리 후 30초 내 복귀했는가
4. 2차 잔소리 후 2분 내 복귀했는가
5. 3차 잔소리 후에도 무시하는가

### 2-4. 예외 상황

- **예외 1: 권한 미부여** — AccessibilityService 권한이 없으면 핵심 루프 자체가 작동 불가. 첫 세션 시작 시 권한 안내 화면으로 강제 분기.
- **예외 2: 3단계 모두 무시** — 사용자가 끝까지 안 돌아오는 케이스. 세션을 "실패"로 기록하고 리포트에 명시 (다음 세션 설계 입력값).

## 3. Wireframe

ASCII 박스로 작성. 색·이미지 X, 구조·정보 배치 O.

### Wireframe ① 홈 화면 (시작 화면)

- **화면 목적:** 사용자가 1초 안에 "지금 세션 시작할지" 결정하게 한다.
- **포함 요소:** 앱 이름 / 사용자 인사 / 어제 복귀율 / 오늘 TODO / 세션 시작 버튼(가장 큰 CTA) / 익명 카운터 / 하단 탭

```
┌──────────────────────────────┐
│ Zan Sol                  [⚙] │
├──────────────────────────────┤
│ 안녕, 우진                    │
│ 어제 복귀율 78%   ↑3%p        │
├──────────────────────────────┤
│ 오늘 할 일                    │
│ • 자료조사 보고서 (D-3) ⚠     │
│ • 발표 슬라이드 초안 (D-7)    │
│ [+ 할 일 추가]                │
├──────────────────────────────┤
│      ┌──────────────┐        │
│      │ 세션 시작 ▶  │        │
│      └──────────────┘        │
├──────────────────────────────┤
│ 지금 경희대생 142명 집중 중   │
├──────────────────────────────┤
│ [홈] [세션] [할일] [리포트]   │
└──────────────────────────────┘
```

- **이동:** [세션 시작] → 세션 설정 / [+ 할 일 추가] → 할 일 화면 / [⚙] → 설정.

---

### Wireframe ② 세션 진행 화면 (핵심 기능 수행)

- **화면 목적:** 진행 중인 세션의 경과 시간과 현재 TODO를 즉시 확인하게 한다. 화면 자체는 단순 — 핵심 동작은 백그라운드. **세션 시간을 미리 정해두지 않는다 (목표 시간 X).** 사용자가 "이제 그만"이라고 판단할 때까지 누적되는 경과 시간만 표시한다.
- **포함 요소:** 큰 경과 타이머 / 현재 TODO 및 마이크로 태스크 / 종료 버튼 / 익명 카운터

```
┌──────────────────────────────┐
│ 집중 중                       │
├──────────────────────────────┤
│                              │
│         00:35:24             │
│        경과 시간              │
│                              │
├──────────────────────────────┤
│ 지금 하고 있는 일             │
│ ▶ 자료조사 보고서             │
│   마이크로 태스크:            │
│   "참고 논문 1편만 정리"      │
├──────────────────────────────┤
│ 142명과 함께 집중 중          │
├──────────────────────────────┤
│         [세션 종료]           │
└──────────────────────────────┘
```

- **이동:** [세션 종료] → 세션 완료 리포트 / 딴짓 감지 시 → 시스템 오버레이로 잔소리 팝업.

---

### Wireframe ③ 잔소리 오버레이 (제품 본체 화면)

- **화면 목적:** 도피 "그 순간"에 사고 흐름을 끊고 복귀를 유도한다. **이 화면이 제품의 본체다.**
- **특수성:** 일반 화면이 아니라 **SYSTEM_ALERT_WINDOW로 다른 앱 위에 그려진다.** 사용자가 명시적으로 닫지 않으면 사라지지 않음.
- **포함 요소:** 단계 표시(1/3, 2/3, 3/3) / 잔소리 문구 / 원래 하던 일 표시 / **돌아가기 버튼(가장 큰 CTA)** / (1·2차에만) 잠깐 닫기

```
─── (인스타 화면 위에) ────────────
┌──────────────────────────────┐
│  ⚡ 잔솔                   1/3│
├──────────────────────────────┤
│                              │
│   "자료 하나만 찾아봐"        │
│                              │
│   지금 하던 일:               │
│   참고 논문 1편만 정리        │
│                              │
│  ┌────────────────────────┐  │
│  │  돌아가기 (집중 모드)  │  │  ← 큰 CTA
│  └────────────────────────┘  │
│                              │
│  [5초 후 잠깐 닫기]           │
└──────────────────────────────┘
```

**3단계 변형:**

| 단계 | 표시 | 문구 톤 | 닫기 옵션 | 심리 메커니즘 |
|---|---|---|---|---|
| 1차 (즉시) | 1/3 | 행동 제안 | "잠깐 닫기" 가능 | 행동활성화 (행동이 동기를 이끈다) |
| 2차 (30초 후) | 2/3 | 공감 + 난이도 낮추기 | "잠깐 닫기" 가능 | 자기효능감 (Bandura) |
| 3차 (2분 후) | 3/3 | 압박 ("내일의 너에게 미안") | **닫기 제거**, 돌아가기 또는 명시적 "그래도 무시"만 | 심리적 저항 회피 (Reactance) |

- **이동:** [돌아가기] → 집중 세션 앱 복귀 (복귀율 카운트) / [잠깐 닫기] → 30초 후 2차 트리거.

---

### Wireframe ④ 세션 완료 리포트 (결과·완료 화면)

- **화면 목적:** 방금 세션의 결과를 즉시 확인하고, 다음 행동을 제시한다. 목표 시간 개념이 없으므로 **달성률 대신 누적 집중 시간 자체와 복귀율**을 핵심 지표로 표시.
- **포함 요소:** 총 집중 시간(경과) / 딴짓 시도 횟수 / 단계별 복귀율 / 다음 행동 제안 / [다음 세션] [홈으로]

```
┌──────────────────────────────┐
│ 세션 완료                     │
├──────────────────────────────┤
│      54분 집중                │
│      (오늘 누적 1시간 42분)   │
├──────────────────────────────┤
│ 딴짓 시도 4회                 │
│  • 1차에서 복귀 3회 ✓        │
│  • 2차에서 복귀 1회 ✓        │
│  • 3차까지 간 경우 0회        │
│                              │
│ 복귀율 100% (지난 주 평균 81%)│
├──────────────────────────────┤
│ 다음에 할 일 (D-3)            │
│ ▶ 자료조사 보고서             │
├──────────────────────────────┤
│ [다음 세션 시작]   [홈으로]   │
└──────────────────────────────┘
```

- **이동:** [다음 세션 시작] → 세션 설정 / [홈으로] → 홈.

### Wireframe ⑤ 설정 — 딴짓 앱 관리 (보조)

- **화면 목적:** 사용자가 직접 freshness 유지. 신규 앱 10분 체류 시 자동 등록 제안.
- **포함 요소:** 등록된 앱 리스트 / 자동 감지된 후보 / 추가·삭제

```
┌──────────────────────────────┐
│ ← 딴짓 앱 관리                │
├──────────────────────────────┤
│ 등록된 앱 (5)                 │
│ ☑ Instagram          [삭제]  │
│ ☑ YouTube            [삭제]  │
│ ☑ TikTok             [삭제]  │
│ ☑ Twitter            [삭제]  │
│ ☑ Reddit             [삭제]  │
├──────────────────────────────┤
│ 최근 자주 사용 (자동 감지)    │
│ • Threads (10분+ 체류 3회)    │
│   [딴짓 앱으로 등록]          │
├──────────────────────────────┤
│ [+ 직접 추가]                 │
└──────────────────────────────┘
```

## 4. Wire Flow

```
[홈]
  ├─ [세션 시작] → [세션 설정] → [세션 진행]
  │                                 ├─ 정상종료 → [세션 완료 리포트]
  │                                 └─ 딴짓 감지 → [잔소리 1차]
  │                                                    ├─ 돌아가기 → [세션 진행]
  │                                                    └─ 무시 30초 → [잔소리 2차]
  │                                                                       ├─ 돌아가기 → [세션 진행]
  │                                                                       └─ 무시 2분 → [잔소리 3차]
  │                                                                                          ├─ 돌아가기 → [세션 진행]
  │                                                                                          └─ 끝까지 무시 → [세션 완료 리포트(실패)]
  ├─ [+ 할 일 추가] → [할 일 화면]
  ├─ [⚙ 설정] → [설정 화면] → [딴짓 앱 관리]
  └─ [리포트 탭] → [리포트 화면]

[세션 완료 리포트]
  ├─ [다음 세션] → [세션 설정]
  └─ [홈으로] → [홈]
```

**Wire Flow 점검:**

- 화면 간 이동 자연스러움: 핵심 루프가 1탭 깊이.
- 다음 행동 제시: 모든 종료 화면(리포트)에 [다음 세션] CTA.
- 불필요 이동 없음: 세션 진행 중 잔소리는 별도 화면이 아니라 시스템 오버레이.
- 예외 처리: 권한 미부여 → 설정 분기 / 끝까지 무시 → 실패 리포트.

## 5. 설명 문서

1. **핵심 사용자**는 관찰자 없이 혼자 공부하다가 스마트폰으로 도피하는 경희대 대학생이며, 자체 서베이(n=34)에서 도피 경험 85.3%로 문제가 충분히 크다.
2. **핵심 문제**는 도피 "그 순간"에 개입할 수단이 없다는 것이다 — 기존 앱(스크린타임은 사후 리포트, Forest는 의지 전제)은 이 순간을 비워둔다.
3. **IA**는 "홈 → 세션"이 1탭 안에 닿도록 핵심 루프를 가장 짧은 경로로 배치하고, 리포트·설정은 보조 메뉴로 분리하여 사용자가 매번 의식적으로 찾을 필요가 없게 했다.
4. **User Flow**에서 가장 중요하게 고려한 행동은 "딴짓 앱 실행 → 1차/2차/3차 잔소리 → 복귀"의 단일 핵심 루프이며, 세 번의 의사결정 지점(각 단계 무시 여부)이 우리의 Primary Metric인 복귀율의 직접 입력값이 된다.
5. **Wireframe**에서 가장 중요하게 배치한 것은 잔소리 오버레이의 **[돌아가기] 버튼을 가장 큰 CTA**로 두고 1·2차에는 닫기를 두되 3차에는 닫기를 제거한 것이다 — 이는 행동활성화 → 자기효능감(Bandura) → 심리적 저항 회피(Reactance)의 단계별 심리 메커니즘을 화면 구조로 옮긴 것이다.
6. 이 설계는 개발 전에 **핵심 루프가 디바이스 로컬에서 완결**됨을 시각적으로 확인시켜 주고(네트워크 지연이 Product Goal에 영향 없음), 4인 팀이 16주 안에 어떤 화면을 누가 만들지 분담 기준을 제공한다.
