# 0428 UX 설계 — Zan Sol V1 (Draft)

## 1. 프로젝트 주제

**Zan Sol (잔솔)** — 집중 세션 중 딴짓을 자동 감지하고, 유저를 긁는 잔소리로 즉시 개입하는 개인용 자기조절 앱 (Android, V1 = 규칙 기반).

**Product Goal:** 사용자가 혼자 공부 중 스마트폰으로 도피할 때 **그 순간 즉시 개입**하여 원래 집중 과업으로 **복귀**하도록 돕는다.

**핵심 사용자:** 관찰자 없이 혼자 공부하는 경희대 대학생 (서베이 n=34에서 도피 경험 85.3%, 잔소리 앱 의향 70%).

---

## 2. Information Architecture

### 2-1. 사용자 유형

V1은 **단일 유형: 개인 사용자(학생 본인)**. 팀/관찰자/관리자 역할 없음 (5-5 결정: 팀 기능 Out of Scope).

### 2-2. 메뉴 구조표

| 상위 메뉴 | 하위 메뉴 | 주요 기능 | 핵심 데이터 | MVP |
|---|---|---|---|---|
| **홈** | 오늘 카드, 세션 시작 버튼, 익명 카운터 | 지금 무엇을 해야 하는지 1초 안에 파악 + 세션 시작 진입점 | 오늘 TODO, 어제 복귀율, 동시 집중자 수 | P0 (P2: 카운터) |
| **세션** | 세션 진행 화면, 잔소리 오버레이 | **딴짓 감지 → 잔소리 개입 → 복귀** (제품 핵심 루프) | 남은 시간, 현재 TODO, 개입 단계, 잔소리 문구 | P0 |
| **할 일** | TODO 목록, 마감 등록, 마이크로 태스크 | 마감 기반 우선순위화 + 복귀 마찰 최소화용 마이크로 태스크 | TODO 텍스트, 마감일(D-N), 마이크로 태스크 | P0 |
| **리포트** | 일별/주별 리포트 | 집중 시간·복귀율 자기 피드백 | 세션 수, 집중 시간, 복귀율, 단계별 복귀율 | P1 |
| **설정** | 딴짓 앱 등록, 잔소리 강도, 학기 일정, 권한 관리 | 사용자가 직접 freshness 유지 (13-2 결정) | 딴짓 앱 리스트, 강도, 학기 일정, AccessibilityService 권한 | P0 |

### 2-3. IA 설계 의도

- **홈 → 세션**이 1탭 안에 닿음. 핵심 루프가 가장 짧은 경로.
- 리포트·설정은 보조 메뉴 (사용 빈도 낮음).
- "잔소리 오버레이"는 **메뉴 구조 외부**에서 트리거됨 — 시스템 권한(AccessibilityService)으로 앱 외부에서도 뜸. IA 표에는 "세션" 하위로 표기하되, 실제로는 OS 레이어 위에 그려진다.
- 비슷한 기능 중복 없음: TODO 등록은 "할 일"에만, 권한 관리는 "설정"에만.

## 3. User Flow

### 3-1. 선택한 핵심 목표

**"집중 세션을 시작하고, 딴짓을 시도했을 때 잔소리 개입을 받아 원래 세션으로 복귀한다."**

이 목표를 고른 이유: Product Goal("그 순간 즉시 개입하여 복귀") 자체가 이 흐름이다. 다른 후보(온보딩, TODO 등록, 리포트 보기)는 보조 흐름.

### 3-2. 흐름도

```
[홈 화면]
  ↓ [세션 시작] 버튼
[세션 시간·TODO 선택]
  ↓ [시작]
◇ AccessibilityService 권한 있는가?
  ├─ NO → [권한 안내 화면] → 설정으로 이동 → 다시 시도   (예외 1)
  └─ YES ↓
[세션 진행 화면 — 타이머 시작, Foreground Service]
  ↓
◇ 딴짓 앱 실행 감지? (Reactive: <0.1초, Proactive: 30초 폴링)
  ├─ NO → 세션 계속 → 정상 종료 → [세션 완료 리포트]
  └─ YES ↓
[1차 잔소리 오버레이 — 행동 제안: "자료 하나만 찾아봐"]
  ↓
◇ 사용자 반응?
  ├─ 복귀 (집중 앱 전환) → [세션 진행 화면 복귀] → 복귀율 기록
  └─ 무시 (30초 경과) ↓
[2차 잔소리 오버레이 — 공감 + 난이도 낮추기: "막막하지? 한 줄만 써봐"]
  ↓
◇ 사용자 반응?
  ├─ 복귀 → [세션 진행 화면 복귀] → 단계별 복귀율 기록
  └─ 무시 (2분 경과) ↓
[3차 잔소리 오버레이 — 압박: "내일의 너한테 미안하지 않을까?"]
  ↓
◇ 사용자 반응?
  ├─ 복귀 → [세션 진행 화면 복귀]
  └─ 계속 무시 → 세션 실패 기록 → [세션 완료 리포트 — 실패 표시]   (예외 2)

[세션 완료 리포트]
  ├─ 집중 시간, 딴짓 시도 횟수, 복귀율 표시
  └─ [다음 세션 시작] 또는 [홈으로]
```

### 3-3. 의사결정 지점 (5개)

1. AccessibilityService 권한 부여 여부
2. 실행 앱이 사용자 등록 딴짓 앱 리스트에 있는가
3. 1차 잔소리 후 30초 내 복귀했는가
4. 2차 잔소리 후 2분 내 복귀했는가
5. 3차 잔소리 후에도 무시하는가

### 3-4. 예외 상황 (2개 이상)

- **예외 1: 권한 미부여** — AccessibilityService 권한 없으면 핵심 루프 자체가 작동 불가. 첫 세션 시작 시 권한 안내 화면으로 강제 분기.
- **예외 2: 3단계 모두 무시** — 사용자가 끝까지 안 돌아오는 케이스. 세션을 "실패"로 기록하고 리포트에 명시 (다음 세션 설계의 입력값).
- **예외 3 (보조): 메신저 등 정당한 사용** — 5-6 결정에서 메신저는 "특수 카테고리". V1은 일단 등록된 딴짓 앱이면 동일 처리, V2 개선 영역으로 표시.

## 4. Wireframe

ASCII로 작성. 색·이미지 X, 구조·정보 배치 O.

### 4-1. Wireframe ① 홈 화면

- **화면 목적:** 사용자가 1초 안에 "지금 세션 시작할지" 결정하게 한다.
- **포함 요소:** 오늘 TODO 요약 / 어제 복귀율 / 세션 시작 버튼 (가장 큰 CTA) / 익명 카운터 / 하단 탭

```
┌──────────────────────────────┐
│ Zan Sol                  [⚙] │
├──────────────────────────────┤
│ 안녕, 우진                    │
│ 어제 복귀율 78%   ↑3%p        │
├──────────────────────────────┤
│ 오늘 할 일 (D-3)              │
│ • 자료조사 보고서 (D-3) ⚠     │
│ • 발표 슬라이드 초안 (D-7)    │
│ [+ 할 일 추가]                │
├──────────────────────────────┤
│                              │
│      ┌──────────────┐        │
│      │ 세션 시작 ▶  │        │
│      └──────────────┘        │
│                              │
├──────────────────────────────┤
│ 지금 경희대생 142명 집중 중   │
├──────────────────────────────┤
│ [홈] [세션] [할일] [리포트]   │
└──────────────────────────────┘
```

- **다음 화면 이동:** [세션 시작] → 세션 설정 화면 / [+ 할 일 추가] → 할일 화면 / [⚙] → 설정.

### 4-2. Wireframe ② 세션 진행 화면

- **화면 목적:** 진행 중인 세션의 남은 시간과 현재 TODO를 즉시 확인하게 한다. 이 화면 자체는 단순 — 핵심 동작은 백그라운드.
- **포함 요소:** 큰 타이머 / 현재 TODO / 진행률 / 일시정지·종료 버튼 / 익명 카운터.

```
┌──────────────────────────────┐
│ 세션 진행 중               [×]│
├──────────────────────────────┤
│                              │
│         24:37                │
│       남은 시간              │
│                              │
│   ▓▓▓▓▓▓░░░░  35분 / 60분    │
├──────────────────────────────┤
│ 지금 하고 있는 일             │
│ ▶ 자료조사 보고서             │
│   마이크로 태스크:            │
│   "참고 논문 1편만 정리"      │
├──────────────────────────────┤
│ 142명과 함께 집중 중          │
├──────────────────────────────┤
│ [일시정지]      [세션 종료]   │
└──────────────────────────────┘
```

- **다음 화면 이동:** [세션 종료] → 세션 완료 리포트 / 딴짓 감지 시 → 시스템 오버레이로 잔소리 팝업.

### 4-3. Wireframe ③ 잔소리 오버레이 (시스템 위에 뜨는 핵심 화면)

- **화면 목적:** 도피 "그 순간"에 사고 흐름을 끊고 복귀를 유도한다. **이 화면이 제품의 본체다.**
- **특수성:** 일반 화면이 아니라 **SYSTEM_ALERT_WINDOW로 다른 앱(인스타·유튜브) 위에 그려진다.** 사용자가 닫지 않으면 안 사라짐.
- **포함 요소:** 단계 표시 / 잔소리 문구 / 복귀 버튼(가장 크게) / 5초 후 닫기 / 마이크로 태스크 힌트.

```
─── (인스타 화면 위에) ────────────
┌──────────────────────────────┐
│  ⚡ 잔솔                   1/3│
├──────────────────────────────┤
│                              │
│   "자료 하나만 찾아봐"        │
│                              │
│   지금 하던 일:               │
│   참고 논문 1편만 정리        │
│                              │
│  ┌────────────────────────┐  │
│  │  돌아가기 (집중 모드)  │  │  ← 큰 CTA
│  └────────────────────────┘  │
│                              │
│  [5초 후 잠깐 닫기]           │
└──────────────────────────────┘
```

**3단계 변형:**

| 단계 | 표시 | 문구 톤 | 닫기 옵션 |
|---|---|---|---|
| 1차 (즉시) | `1/3` | 행동 제안 | "5초 후 닫기" 가능 |
| 2차 (30초) | `2/3` | 공감 + 난이도 낮추기 | "5초 후 닫기" 가능 |
| 3차 (2분) | `3/3` | 압박 (내일의 너에게 미안) | 닫기 X, **돌아가기 또는 명시적 "그래도 무시"만** |

- **다음 화면 이동:** [돌아가기] → 집중 세션 앱으로 복귀 (복귀율 카운트) / [잠깐 닫기] → 30초 후 2단계 트리거.

### 4-4. Wireframe ④ 세션 완료 리포트

- **화면 목적:** 방금 세션의 결과를 사용자가 즉시 확인하고, 다음 행동을 제시한다.
- **포함 요소:** 총 집중 시간 / 딴짓 시도 횟수 / 단계별 복귀율 / 다음 세션 / 홈.

```
┌──────────────────────────────┐
│ 세션 완료                     │
├──────────────────────────────┤
│                              │
│      54분 집중                │
│      목표 60분 중 90%         │
│                              │
├──────────────────────────────┤
│ 딴짓 시도 4회                 │
│  • 1차 잔소리에서 복귀 3회 ✓ │
│  • 2차 잔소리에서 복귀 1회 ✓ │
│  • 3차까지 간 경우 0회        │
│                              │
│ 복귀율 100% (지난 주 평균 81%)│
├──────────────────────────────┤
│ 다음에 할 일 (D-3)            │
│ ▶ 자료조사 보고서 — 50% 진행  │
├──────────────────────────────┤
│ [다음 세션 시작]   [홈으로]   │
└──────────────────────────────┘
```

- **다음 화면 이동:** [다음 세션 시작] → 세션 설정 화면 / [홈으로] → 홈.

### 4-5. Wireframe ⑤ 설정 — 딴짓 앱 등록 (보조)

- **화면 목적:** freshness의 사용자 주도 유지 (13-2). 신규 앱 10분 체류 시 자동 등록 제안 처리.
- **포함 요소:** 등록된 앱 리스트 / 자동 감지된 후보 / 추가·삭제.

## 5. Wire Flow

```
[홈]
  ├─ [세션 시작] → [세션 설정] → [세션 진행]
  │                                    ├─ 정상종료 → [세션 완료 리포트]
  │                                    └─ 딴짓 감지 → [잔소리 오버레이 1차]
  │                                                       ├─ 돌아가기 → [세션 진행]
  │                                                       └─ 무시 30초 → [잔소리 2차]
  │                                                                          ├─ 돌아가기 → [세션 진행]
  │                                                                          └─ 무시 2분 → [잔소리 3차]
  │                                                                                            ├─ 돌아가기 → [세션 진행]
  │                                                                                            └─ 끝까지 무시 → [세션 완료 리포트(실패)]
  ├─ [+ 할 일 추가] → [할 일 화면]
  ├─ [⚙ 설정] → [설정 화면] → [딴짓 앱 관리]
  └─ [리포트 탭] → [리포트 화면]

[세션 완료 리포트]
  ├─ [다음 세션] → [세션 설정]
  └─ [홈으로] → [홈]
```

**Wire Flow 점검:**
- 화면 간 이동 자연스러움: 핵심 루프가 1탭 깊이.
- 다음 행동 제시: 모든 종료 화면(리포트)에 [다음 세션] CTA 있음.
- 불필요 이동 없음: 세션 진행 중 잔소리는 별도 화면이 아니라 오버레이.
- 예외 상황 처리: 권한 미부여 → 설정 분기, 끝까지 무시 → 실패 리포트.

## 6. 설명 문서 (5~7문장)

1. **핵심 사용자**는 관찰자 없이 혼자 공부하다가 스마트폰으로 도피하는 경희대 대학생이며, 서베이 n=34에서 도피 경험 85.3%로 문제가 충분히 크다.
2. **핵심 문제**는 도피 "그 순간"에 개입할 수단이 없다는 것이다 — 기존 앱(스크린타임은 사후, Forest는 의지 전제)은 이 순간을 비워둔다.
3. **IA**는 "홈 → 세션"이 1탭 안에 닿도록 핵심 루프를 가장 짧은 경로로 배치하고, 리포트·설정은 보조 메뉴로 분리하여 사용자가 매번 의식적으로 찾을 필요가 없게 했다.
4. **User Flow**에서 가장 중요하게 고려한 행동은 "딴짓 앱 실행 → 1차/2차/3차 잔소리 → 복귀"의 단일 핵심 루프이며, 세 번의 의사결정 지점(각 단계 무시 여부)이 우리 Product Goal("복귀") 측정의 직접 입력값이 된다.
5. **Wireframe**에서 가장 중요하게 배치한 것은 잔소리 오버레이의 **[돌아가기] 버튼을 가장 큰 CTA**로 두고, 1·2차에는 닫기 옵션을 두되 3차에는 닫기를 제거한 것이다 — 이는 5-6 에스컬레이션 심리 메커니즘(행동제안 → 공감 → 압박)을 화면 구조로 그대로 옮긴 것이다.
6. 이 설계는 개발 전에 **핵심 루프가 디바이스 로컬에서 완결됨을 시각적으로 확인**시켜 주고(네트워크 지연이 Product Goal에 영향 없음을 IA·Flow에서 모두 확인), 4인 팀이 16주 안에 어떤 화면을 누가 만들지 분담 기준을 제공한다.

## 7. 수업 원칙 매핑

- **Step 1 (Product Goal 먼저):** Wireframe 모든 화면이 "복귀"라는 Primary Metric에 봉사. 잔소리 오버레이가 그 직접 측정점.
- **Step 5 (Simple Solution):** IA에 V2/팀 기능 흔적 없음. P0/P1만 화면화. P2(익명 카운터)는 홈 한 줄로만.
- **Step 6 (알고리즘이 되어보기):** User Flow의 3단계 잔소리 분기가 학습상담사·교사 인터뷰의 개입 절차를 그대로 옮긴 것 (5-6).
- **Step 8 (framing 변경):** "개인화 추천" 대신 "고정 문구 + 단계별 분기"로 framing해서 V1에서 화면 구현 가능.
- **수업 핵심 메시지 "기능을 많이 넣는 것보다 핵심 흐름을 명확히":** 메뉴 5개로 압축, Wireframe 5개 모두 핵심 루프에 봉사.

## 8. 우리 팀의 최종 판단

**핵심 루프(딴짓 감지 → 잔소리 → 복귀)의 시각화가 이번 설계의 본질이다.** 일반적인 UX 과제는 메뉴 구조나 사용성에 무게가 가지만, Zan Sol은 **사용자가 우리 앱 안에 머무르지 않는 시간(=딴짓 앱 사용 중)에 우리 화면이 위에 떠야** 하는 특이성을 가진다. 이 사실을 IA(잔소리 오버레이를 메뉴 외부 트리거로 표기)와 Wireframe(시스템 오버레이를 별도 항목으로)에서 명시적으로 드러낸 것이 다른 팀과 차별점.
