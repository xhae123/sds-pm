# 0428 In-Class Assignment — UX 설계 (IA · User Flow · Wireframe)

- **날짜:** 2026-04-28
- **유형:** UX 설계 실습 (신규 포맷 — 기존 catalog에 없음)
- **대상 프로젝트:** Zan Sol (V1 — 규칙 기반 잔소리 개입 앱)

## 목적

개발 직전 단계에서 서비스의 **구조 / 사용자 흐름 / 화면 구성**을 명시화한다.
"기능을 많이 넣는 것보다 핵심 흐름을 명확히" 하는 것이 평가 핵심.

## 제출 항목

| 항목 | 필수 | 내용 |
|---|---|---|
| Information Architecture | 필수 | 메뉴 구조표 (사용자 유형 / 상위·하위 메뉴 / 주요 기능 / 핵심 데이터) |
| User Flow | 필수 | 핵심 사용자 목표 1개의 흐름도 (시작·행동·입력·**의사결정**·**예외 1+**·완료) |
| Wireframe | 필수 | 핵심 화면 **3개 이상** (시작/홈 + 핵심기능 + 결과/완료) |
| Wire Flow | 권장 | Wireframe 간 이동 흐름 |
| 설명 문서 | 필수 | 5~7문장. 핵심사용자/핵심문제/IA정리방식/Flow핵심행동/Wireframe핵심배치/설계효용 |

## 화면별 필수 작성 요소

화면 이름 / 화면 목적 / 주요 정보 영역 / 입력 항목 / 버튼 / 다음 화면 이동 동작 / 사용자가 확인할 결과 정보

## 평가 기준 (7개)

문제 적합성 · IA 명확성 · User Flow 논리성 · Wireframe 완성도 · 사용자 중심성 · 실현 가능성 · 설명력

## 주의사항

- Wireframe은 디자인 완성도 X, **구조·정보 배치** O
- User Flow에 **의사결정 지점 + 예외 1개 이상** 반드시 포함
- IA에서 메뉴명은 사용자 언어로
- MVP(이번 학기 V1 = P0/P1) vs 추후 기능(P2/V2) 구분
- 각 화면이 "사용자가 무엇을 해야 하는가"에 답해야

## 도구

Markdown 텍스트 + ASCII 다이어그램 (PDF 예시도 ASCII 박스 사용). Figma/Miro 불필요.

작성 중.
