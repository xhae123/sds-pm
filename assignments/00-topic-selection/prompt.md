## 과제 목적

아이디어를 실제 프로젝트 주제로 구체화하는 첫 단계.
> "현실의 문제를 찾고 → 어플리케이션 형태로 기획하고 → 머신러닝 요소를 연결하고 → 최종 결과물까지 구현하는 과정 전체"

## 주제 선정 3원칙

1. **가까이 있는 문제** — 거창한 주제 X, 실생활 불편함 O
2. **익숙한 문제** — 상황·맥락·시스템 작동 방식까지 익숙해야
3. **전문가 해결 프로세스를 떠올릴 수 있을 것** — 인터뷰·벤치마킹·상상이라도

## 머신러닝 적용 (중요 — V1/V2 분리의 출처)

> "최종 프로젝트에 반드시 머신러닝이 포함되어야만 하는 것은 아닙니다."
> - **Version 1:** 머신러닝 없이 작동하는 기본 어플리케이션
> - **Version 2:** 머신러닝 요소가 추가된 확장 버전
> "ML이 최종 결과물에 포함되지 않더라도 다른 부분이 충실하게 구현되어 있다면 감점하지 않겠습니다."

→ **우리 Zan Sol의 V1(규칙)/V2(ML) 구조 자체가 이 한 줄에서 시작됨.**

## 할 일

1. 팀별 프로젝트 주제 1개 선정
2. AI 활용 기획안 초안 작성

## 기획안 5항목

| 항목 | 내용 |
|---|---|
| 해결하려는 문제 | 어떤 현실 문제 / 누가·어떤 상황 |
| 만들고 싶은 어플리케이션 | 어떤 기능 / 사용자가 어떻게 사용 |
| 문제 해결 절차 (Procedure) | 입력 → 시스템 내부 절차 → 1·2·3단계 |
| 머신러닝 요소 | 어느 단계에 ML 필요 / 어느 단계는 절차적 코딩 |
| 전문가 벤치마킹 | 누가 잘 푸는가 / 그 전문가의 판단 기준 |

## AI 활용 시 4원칙 (시험 출제 가능)

1. **AI 답에 내 아이디어가 끌려가면 안 됨**
2. **프롬프트에 맥락을 충분히** — **R-O-C-O** (Role / Objective / Context / Output)
3. **일반적 앱 X, 우리 팀만의 포인트 O**
4. (지시서 외 함의) AI 결과는 검토 대상이지 따라갈 답이 아니다

## 평가 기준

- 수업 내용 실제 적용 여부
- 문제 구체적 정의
- 팀 의도 유지한 채 구현 시도
- 시행착오 진지하게 탐색
