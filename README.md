# sds-pm

서데사 팀 프로젝트(Zan Sol)의 팀 단위 AI 운영 레포.

## 푸는 문제

조별과제에서 AI가 주는 힘의 대부분이 사라지는 지점은 "AI가 좋지 않아서"가 아니라 **팀이 AI를 팀 단위로 쓰지 못하기 때문**이다.

- 팀원마다 쓰는 AI가 다르다 (ChatGPT, Claude, Gemini).
- 모두 에이전트가 아니라 챗봇으로 쓴다. 맥락은 대화창에서만 유효하고, 창을 닫으면 사라진다.
- 팀 소통은 카톡에서 이뤄지는데, 어떤 AI도 그 카톡을 읽지 못한다.

결과: 조원 4명이 각자의 AI에 같은 프로젝트 맥락을 4번 다르게 설명하고, 산출물 4개가 따로 논다. 발표 전날 누군가 "내가 통일할게"로 수습한다.

## 이 레포가 하는 것

**팀의 프로젝트 맥락을 하나의 문서 묶음으로 고정해, 누가 어떤 AI를 쓰든 동일한 출발점에서 작업하게 만든다.**

카톡을 대체하지 않는다. 카톡은 그대로 두고, 그 위에 "한 번 복붙하면 AI가 우리 팀을 이해하게 되는 맥락 레이어"를 얹는다. 의사결정·근거·수업 프레임워크·과제 포맷·과거 피드백이 이 레이어에 구조화되어 있다.

결과:

- 팀원이 새 AI 대화를 열 때마다 프로젝트를 다시 설명할 필요가 없다.
- 누구의 AI든 같은 수치·같은 결정·같은 프레임을 전제로 답한다.
- 과제별 초안이 수업 프레임워크에 매핑된 상태로 생성된다.
- 과거 피드백이 다음 과제 작성 시 자동으로 참조된다.

추가로 PM 환경에서는 에이전트가 서데사 단톡에 직접 메시지를 보낼 수도 있다 (카카오톡 Mac 앱 UI 자동화, PM 승인 필수).

## 사용법

### 챗봇 사용자 (ChatGPT, Claude 웹, Gemini 등)

1. 새 대화에서 다음 파일을 순서대로 붙여넣는다.
   - `CONTEXT.md`
   - `MEMORY.md`
   - `course/` 하위 4개 파일
2. "서데사 팀 AI PM으로 행동하라"고 지시한다.
3. 이후 질문하거나 과제 PDF를 첨부한다.

ChatGPT Projects 또는 Claude Projects 기능으로 위 묶음을 고정해두면 재붙여넣기가 필요 없다.

### Claude Code 사용자

```bash
git clone https://github.com/xhae123/sds-pm.git
cd sds-pm
claude
```

자동 로드되는 슬래시 명령:

- `/run-assignment <PDF 경로>` — 과제 초안을 draft → review → final 3단계로 생성
- `/update-context <정보>` — 새 정보를 기존 문서와 비교·충돌 점검 후 반영 제안
- `/kakao-team` — 서데사 단톡 초안 작성 및 PM 승인 후 발송

## 구조

```
AGENTS.md · CLAUDE.md       AI 진입점
CONTEXT.md                  프로젝트 마스터 문서
MEMORY.md                   축적 메타 레슨
course/                     수업 프레임워크 정제
assignments/                과제 이력 (draft → review → final)
.claude/agents/             PM 페르소나
.claude/skills/             슬래시 명령 정의
```

## 운영 규칙

- AI 초안을 그대로 제출하지 않는다. 모든 과제는 draft → review → final 3단계를 거치며, 최종안에 "AI 초안 대비 개선점" 섹션이 포함된다.
- `CONTEXT.md`의 수치는 모두 근거가 있다. 변경 시 근거 제시 후 PM 승인이 필요하다.
- 이번 학기 결과물은 V1이다. V2 논의는 답안 말미 한 줄로 제한한다.
- 교수 PDF, 카톡 원본, 발송 전 초안은 로컬에만 보관한다 (`.gitignore`).

## 업데이트

PM이 주기적으로 push한다. 팀원은 `git pull`로 동기화한다. 새 정보가 생기면 PM에게 공유하면 `/update-context`가 기존 내용과의 충돌·중복·근거 정합성을 점검한 뒤 반영 계획을 제시한다.
